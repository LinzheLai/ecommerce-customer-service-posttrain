# 电商客服训练数据预处理说明

## 1. 目标

数据预处理的目标是将原始电商客服对话数据转换为结构统一、文本规范、可直接用于 SFT 和 DPO 训练的高质量样本。

核心工作包括：

- 统一数据格式
- 清洗脏文本
- 规范对话角色
- 稳定回复边界
- 过滤低质量样本
- 适配多种训练格式

---

## 2. 原始数据问题

原始数据虽然已整理为 `instruction / input / output` 结构，但仍存在以下问题：

- 标点、空格、换行不规范
- 对话角色表达不统一
- `input` 与 `output` 边界不稳定
- 存在占位符、乱码、异常符号等噪声
- 部分样本存在角色错位或语义断裂

---

## 3. 预处理流程

### 3.1 数据读取

支持读取 `json` 和 `jsonl` 数据，并对字段完整性与基本合法性进行检查。

### 3.2 基础文本清洗

统一处理以下内容：

- 统一换行符
- 去除 BOM 等特殊字符
- 去除多余空格和空行
- 统一中英文标点
- 压缩重复换行

### 3.3 任务前缀处理

对固定任务提示语进行识别和规范，避免在不同字段中重复保留相同说明。

### 3.4 对话解析

将 `input` 中的历史对话从纯文本解析为结构化轮次，依据 `用户：` 和 `客服：` 分割对话内容。

### 3.5 角色统一

将原始数据中不统一的角色表达统一映射为标准角色：

- 用户侧统一为 `user`
- 客服侧统一为 `assistant`

### 3.6 回复边界稳定化

统一保证上下文以 `客服：` 作为回复起点，`output` 仅保留客服回复正文，避免角色前缀重复进入监督目标。

### 3.7 输出清洗

对 `output` 做进一步清洗，包括：

- 去除前后空白
- 去除意外残留的角色前缀
- 过滤空回复

### 3.8 脏样本过滤

过滤以下低质量样本：

- 空样本
- 极短无效样本
- 乱码或异常文本
- 角色错位样本
- 明显跑题或语义混乱样本

### 3.9 去重处理

对重复或高度相似样本进行去重，减少无效重复数据。

---

## 4. 输出格式

根据训练阶段需要，预处理后的数据可转换为以下格式。

### 4.1 SFT 三字段格式

```json
{
  "instruction": "...",
  "input": "...",
  "output": "..."
}
```

### 4.2 prompt / completion 格式
```json
{
  "prompt": [
    {"role": "system", "content": "..."},
    {"role": "user", "content": "..."}
  ],
  "completion": [
    {"role": "assistant", "content": "..."}
  ]
}
```

###  4.3 messages 格式
```json
{
  "messages": [
    {"role": "system", "content": "..."},
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."}
  ]
}
```

###  4.4 DPO 偏好格式
```json
{
  "instruction": "...",
  "input": "...",
  "chosen": "...",
  "rejected": "..."
}
```

## 5. 数据质量提升要点
数据质量提升主要通过以下方式实现：

- 统一样本结构，降低格式噪声
- 规范文本内容，减少分词不稳定性
- 明确回复边界，增强监督信号
- 清除脏样本，提升训练样本纯度
- 适配不同训练格式，保证训练链路一致性

## 6. 总结
数据预处理阶段重点解决了原始客服对话数据中存在的格式不统一、文本不规范、角色混乱、边界不清晰和脏样本等问题，为后续 SFT 和 DPO 训练提供了稳定、规范、可复现的数据基础。
